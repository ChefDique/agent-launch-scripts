# AgentRemote Quality Gates

Task: ALS-QUALITY-003

This matrix defines the minimum regression coverage required before AgentRemote,
launcher, tmux/iTerm, pet, voice, registry, sidecar, or lifecycle-hook changes
can be called complete. It complements
`docs/operations/agentremote-operator-contract.md`: static and isolated tests
are the default evidence layer; live desktop checks require Richard's explicit
approval in the current turn.

## Gate Rules

- Treat AgentRemote as a local operator HUD over the Swarmy-owned runtime path.
  Do not prove behavior by mutating Richard's live iTerm, tmux, AgentRemote, or
  AppleScript desktop state unless he approved that exact live check.
- Use static tests for policy, command shape, registry projection, IPC
  validation, renderer wiring, and parser/filter behavior.
- Use isolated tests for throwaway tmux sessions, mocked IPC, mocked filesystem
  registries, fake sidecars, local fixture transcripts, and non-live Electron
  helpers.
- Live checks are approval-gated and must report the selected agent/runtime,
  effective command, pane/window identity, process path, layout materialization,
  and any cleanup residue.
- A green unit test is not a live summon PASS. It only reduces the blast radius
  before an approved live check.

Blast radius: Codex/Hermes/OpenClaw receive stray blank Enter or Claude receives missing dangerous-permission confirmation. | `test/launch-agent-runtime.test.sh` plus shell/static checks around `launch-agent.sh` must prove `startup_injection.include/exclude` policy gates `dangerous_permission_enter` and `startup_lines`, and that argv arrays are used. | Run launcher tests with fixture registry rows for Claude opt-in, Claude excluded policy, Codex, Hermes, and OpenClaw. | Option-key shortcuts | Option-modified shortcuts can be swallowed by Electron/iTerm/tmux, interpreted as escape sequences, or conflict with OS/app shortcuts. | Pane / window naming | Display names, pane tags, sidecar entries, and tmux titles can diverge. | Pet chat | Terminal chrome, prompt hints, statuslines, unrelated panes, or stale transcripts can appear as chat. | Voice / STT | Local Whisper can fail silently, send partial transcript, target the wrong agent, or regress hold-to-talk feedback. | Sidecar / message-agent | Message delivery can use stale configured coordinates instead of live pane ids, or ACK/parrot traffic can masquerade as useful routing. | Hooks / cleanup | Lifecycle hooks or cleanup can block valid same-repo work, allow cross-repo/live-surface mutation, erase useful caches/state, or leave stale Electron/tmux/iTerm residue. ## Completion Standard

For any touched surface, final closeout must state:

- changed files and ownership boundary,
- static tests run,
- isolated tests run,
- live checks skipped or completed, with approval status,
- remaining blast radius, if any,
- dirty files that predated or remain outside the task.
